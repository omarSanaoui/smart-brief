React Router

Everything about client-side routing in React — from basic <Route> setup to NavLink styling, nested routes, protected routes, and every hook available. All examples come from the Smart Brief codebase.

What is React Router?

React Router is the standard routing library for React. It lets you build a Single Page Application (SPA) where the URL changes and different components are shown, but the browser never makes a full page reload.

Without React Router, your React app is just one page. React Router maps URL paths to components — when the user navigates to /my-briefs, React Router renders the MyBriefs component without fetching a new HTML file from the server.

SPA vs Traditional Website

Traditional Website (MPA)
User clicks a link →
Browser sends GET /about.html to server →
Server sends back a new full HTML page →
Browser discards current page, renders new one →
Flash / white screen between pages
React SPA with React Router
User clicks a Link →
React Router intercepts the click →
URL updates (history.pushState) →
React renders a different component →
No server round-trip, no flash

The server only serves index.html once. After that, all navigation is handled by JavaScript in the browser. The back/forward buttons still work because React Router uses the browser's History API.

Setup & BrowserRouter

React Router must wrap your entire app. This is done in main.tsx, wrapping the root component with <BrowserRouter>: 

main.tsx
import { BrowserRouter } from "react-router-dom";
import { Provider }    from "react-redux";
import { store }       from "./app/store";
import App             from "./App";

createRoot(document.getElementById("root")!).render(
  <Provider store={store}>
    <BrowserRouter>   <!-- ← ALL routing hooks work inside here -->
      <App />
    </BrowserRouter>
  </Provider>
);
Order matters BrowserRouter provides the routing context that all hooks like useNavigate, useLocation, and useParams read from. If you try to use those hooks outside of BrowserRouter, you'll get an error. Redux's Provider and BrowserRouter can be in either order — they are independent.

Router Types

RouterURL styleWhen to use
BrowserRouter example.com/briefs/123 Standard web apps. Needs a server that serves index.html for all paths. Used in Smart Brief.
HashRouter example.com/#/briefs/123 Static file hosting with no server config (GitHub Pages). The # part is never sent to the server.
MemoryRouter No URL change Testing environments, React Native (no browser URL bar).
createBrowserRouter Same as BrowserRouter React Router v6.4+ data API (loaders, actions). More advanced, not used in Smart Brief.

Routes & Route

<Routes> is the container. It looks at the current URL and renders the first <Route> whose path matches. Only one route renders at a time (unlike the old v5 <Switch> — same concept, new name). 

import { Routes, Route } from "react-router-dom";

<Routes>
  <Route path="/"             element={<Home />}         />
  <Route path="/briefs"       element={<BriefsDashboard />} />
  <Route path="/briefs/:id"   element={<BriefDetails />}    />
  <Route path="/login"        element={<Login />}          />
  <Route path="*"             element={<NotFound />}        />  {/* catch-all */}
</Routes>

Path patterns

PatternMatchesExample
/briefsExact path only/briefs
/briefs/:idPath with a dynamic segment/briefs/abc-123
/user/:id/posts/:postIdMultiple dynamic segments/user/5/posts/42
/files/*Wildcard — matches anything after/files/a/b/c
*Catch-all — matches everything not matched aboveUsed for 404 pages
React Router v6 is exact by default In the old v5, you had to add the exact prop to prevent / from matching every route. In v6 (what Smart Brief uses), all routes are exact by default. /briefs will NOT match /briefs/123. You only need /* for wildcard matching.

Link

<Link> renders an <a> tag in the DOM but intercepts the click — preventing a full page reload and using React Router's history-based navigation instead.

❌ Regular anchor — causes full reload
<a href="/login">Log In</a>
// Sends a GET request to the server
// Browser reloads — Redux state is wiped
✅ React Router Link — no reload
<Link to="/login">Log In</Link>
// Just changes the URL in the browser bar
// React Router renders <Login />
// Redux state is preserved
Navbar.tsx
import { Link } from "react-router-dom";

// Logo links to home
<Link to="/" className="shrink-0">
  <Logo className="w-44" />
</Link>

// Auth links in the navbar
<Link to="/login"    className="text-white/80 hover:text-sbteal ...">Log In</Link>
<Link to="/register" className="bg-sbpurple px-6 py-2 ...">Register</Link>

NavLink

<NavLink> is a special version of <Link> that knows whether its destination is the currently active route. It automatically adds an active class (and/or pending class for loading states). This is what Smart Brief uses for the navbar.

The key difference from <Link> is that className and style accept a function instead of a static value:

// Link — static className
<Link to="/briefs" className="my-class">Briefs</Link>

// NavLink — function className that receives { isActive, isPending }
<NavLink
  to="/briefs"
  className={({ isActive }) => isActive ? "active-class" : "normal-class"}
>
  Briefs
</NavLink>

NavLink — Advanced Details

The end prop — controlling what "active" means

By default, <NavLink to="/"> will be active not just on / but also on /briefs, /login, and every other route — because they all start with /. The end prop tells NavLink to only be active when the URL is an exact match: 

// Without end: "/" is active on ALL routes
<NavLink to="/" className={({ isActive }) => isActive ? "active" : ""}>Home</NavLink>

// With end: "/" is only active when URL is exactly "/"
<NavLink to="/" end className={({ isActive }) => isActive ? "active" : ""}>Home</NavLink>
Smart Brief uses "/" without end In Navbar.tsx, the Home NavLink does not have the end prop. This means "Home" is highlighted as active on ALL pages. This works for Smart Brief because the Home link is just a logo/brand link — but for a navigation menu where you want only the right item highlighted, you'd add end to the Home route.

NavLink default active class

Even if you don't pass a className function, <NavLink> automatically adds an active CSS class to the rendered <a> element when it's active. You can target it with plain CSS: 

/* In your CSS file — no className prop needed */
nav a { color: rgba(255,255,255,0.6); }
nav a.active { color: #62d2b2; border-bottom: 2px solid #62d2b2; }

/* That's it — no function needed in JSX */

Navigate Component

<Navigate> is a component that redirects when rendered. It's the declarative version of useNavigate() — useful inside JSX for conditional redirects: 

import { Navigate } from "react-router-dom";

// If user is already logged in, redirect away from the login page
function LoginPage() {
  const isLoggedIn = useAppSelector(selectIsLoggedIn);

  if (isLoggedIn) {
    return <Navigate to="/" replace />;
    // replace=true: replaces current history entry
    // Back button won't go back to the login page
  }

  return <form>...</form>;
}
PropTypeEffect
tostringWhere to redirect
replacebooleanReplace current history entry (default: false — pushes)
stateanyPass state to the destination route

Outlet — Nested Routes

<Outlet> is a placeholder where child routes render. It enables nested layouts — a shared layout component (sidebar, navbar, etc.) wraps multiple pages. 

// Route definition — Dashboard is the layout, children render inside it
<Routes>
  <Route path="/admin" element={<AdminLayout />}>
    <Route index                     element={<AdminHome />}    />  {/* /admin */}
    <Route path="briefs"             element={<AdminBriefs />}  />  {/* /admin/briefs */}
    <Route path="briefs/:id"         element={<BriefDetail />}  />  {/* /admin/briefs/123 */}
    <Route path="employees"          element={<Employees />}    />  {/* /admin/employees */}
  </Route>
</Routes>

// AdminLayout component — the shared wrapper
function AdminLayout() {
  return (
    <div className="flex">
      <AdminSidebar />   {/* always visible */}
      <main>
        <Outlet />       {/* ← child route renders HERE */}
      </main>
    </div>
  );
}

Smart Brief doesn't currently use Outlet — all routes are flat in App.tsx. But it's the standard way to build dashboard layouts.

useNavigate

useNavigate returns a function for programmatic navigation — navigating in response to logic (form submit, button click, after a thunk succeeds), not just a link click. 

import { useNavigate } from "react-router-dom";

const navigate = useNavigate();

// Navigate to a path
navigate("/dashboard");

// Navigate back (like browser back button)
navigate(-1);

// Navigate forward
navigate(1);

// Replace current history entry (back button skips this page)
navigate("/home", { replace: true });

// Navigate with state (readable on the destination with useLocation)
navigate("/verify-code", {
  state: { email: "user@example.com", rememberMe: true }
});

navigate(-1) in Smart Brief

BriefDetails.tsx
// Back button — goes to whatever page the user came from
<button onClick={() => navigate(-1)}>
  <ArrowLeft size={16} /> Back
</button>

// After deleting a brief, go back
const handleDelete = async () => {
  if (!brief || !window.confirm("Are you sure?")) return;
  await dispatch(deleteBriefThunk(brief.id));
  navigate(-1); // back to the list
};

useParams

useParams reads the dynamic segments from the current URL. If your route is /briefs/:id and the user visits /briefs/abc-123, then id is "abc-123". 

import { useParams } from "react-router-dom";

// Route: /briefs/:id
const { id } = useParams<{ id: string }>();
//  ↑ TypeScript generic — tells TS what params to expect
BriefDetails.tsx
export default function BriefDetails() {
  const { id } = useParams<{ id: string }>();
  const dispatch = useAppDispatch();

  useEffect(() => {
    if (id) dispatch(fetchBriefByIdThunk(id));
  }, [id, dispatch]);
  // When the URL is /briefs/abc-123, id = "abc-123"
  // The thunk fetches GET /briefs/abc-123
}

Multiple params

// Route: /users/:userId/briefs/:briefId
const { userId, briefId } = useParams<{ userId: string; briefId: string }>();
useParams always returns strings URL parameters are always strings. If you need a number, convert it: const numId = Number(id). And always check for undefined — a param is undefined if the current route doesn't have that segment.

useLocation

useLocation returns the current location object. It gives you everything about the current URL plus any state that was passed when navigating. 

import { useLocation } from "react-router-dom";

const location = useLocation();

console.log(location.pathname); // "/briefs/abc-123"
console.log(location.search);   // "?token=XYZ&tab=details" (query string)
console.log(location.hash);     // "#section-2" (anchor)
console.log(location.state);    // { email: "...", rememberMe: true } (passed via navigate)
console.log(location.key);      // unique key for this history entry

Reading location.pathname to detect page type

App.tsx
const location = useLocation();

// Check if we're on an auth page to swap the background illustration
const authPaths = ['/register', '/login', '/verify-code', '/forgot-password', '/reset-password'];
const isAuthPage = authPaths.includes(location.pathname);

// Now isAuthPage drives which SVG background is shown
const backgroundAbstract = isAuthPage
  ? <AbstractObj2 ... />
  : <AbstractObj1 ... />;

useSearchParams

useSearchParams reads and writes query string parameters — the ?key=value part of the URL. It works exactly like useState but backed by the URL. 

import { useSearchParams } from "react-router-dom";

const [searchParams, setSearchParams] = useSearchParams();

// READ a param: URL = /reset-password?token=abc123
const token = searchParams.get("token");  // "abc123"
const tab   = searchParams.get("tab");    // null (not in URL)

// CHECK if a param exists
searchParams.has("token");  // true

// SET a param — updates the URL without navigating
setSearchParams({ tab: "settings" });
// URL becomes: /current-page?tab=settings

// REMOVE a param (keep others)
const next = new URLSearchParams(searchParams.toString());
next.delete("token");
setSearchParams(next, { replace: true }); // replace avoids adding to history

ResetPassword — reading a token from the URL

ResetPassword.tsx
// URL: /reset-password?token=abcdef123456
// The token was emailed to the user. They clicked the link.

const [searchParams] = useSearchParams();
const token = useMemo(() => searchParams.get("token")?.trim() || "", [searchParams]);

// Later in handleSubmit:
if (!token) {
  setFieldError("This reset link is invalid or missing.");
  return;
}
await api.post("/auth/reset-password", { token, newPassword: password });

Google OAuth redirect — removing a token from the URL

App.tsx
// After Google login, backend redirects to: /?token=eyJhbGci...
// We need to save the token and REMOVE it from the URL (it's sensitive)

const [searchParams, setSearchParams] = useSearchParams();

useEffect(() => {
  const googleToken = searchParams.get("token");
  if (googleToken) {
    localStorage.setItem("token", googleToken);
    dispatch(setToken(googleToken));

    // Remove ?token= from URL without adding to history
    const next = new URLSearchParams(searchParams.toString());
    next.delete("token");
    setSearchParams(next, { replace: true });
    // URL changes from /?token=eyJ... to / cleanly
  }
}, [dispatch, searchParams, setSearchParams]);

useMatch & useRouteError

useMatch

Returns match information if the given pattern matches the current URL, or null if it doesn't. Useful when you need match details without being inside the matched route.

import { useMatch } from "react-router-dom";

// Returns match object if URL is /briefs/something, null otherwise
const match = useMatch("/briefs/:id");

if (match) {
  console.log(match.params.id); // the brief's id
}

useRouteError

Used inside error boundary routes (React Router v6.4+). When a route throws an error, useRouteError reads it so you can display a nice error page.

import { useRouteError } from "react-router-dom";

function ErrorPage() {
  const error = useRouteError() as { status?: number; statusText?: string };
  return <h1>{error.status}: {error.statusText}</h1>;
}

Nested Routes — Full Example

Nested routes are used to build layouts where part of the page is shared and part changes. The classic use case: a dashboard with a sidebar. 

// App.tsx — define the nested structure
<Routes>
  <Route path="/"       element={<Landing />} />
  <Route path="/login"   element={<Login />}   />

  {/* Dashboard layout wraps all admin pages */}
  <Route path="/admin" element={<DashboardLayout />}>
    <Route index              element={<AdminHome    />} />  {/* GET /admin */}
    <Route path="briefs"     element={<BriefList    />} />  {/* GET /admin/briefs */}
    <Route path="briefs/:id" element={<BriefDetail  />} />  {/* GET /admin/briefs/123 */}
    <Route path="team"       element={<TeamPage     />} />  {/* GET /admin/team */}
  </Route>
</Routes>

// DashboardLayout.tsx — shared wrapper
function DashboardLayout() {
  return (
    <div style={{ display: "flex" }}>
      <Sidebar />      {/* stays visible on all /admin/* pages */}
      <main>
        <Outlet />     {/* AdminHome / BriefList / BriefDetail renders here */}
      </main>
    </div>
  );
}
index route An <Route index /> (no path) renders when the parent's path is matched exactly. In the example above, visiting /admin renders <AdminHome /> through the index route — not /admin/briefs.

Protected Routes

A protected route is a wrapper that redirects unauthenticated users away before rendering the actual page. Smart Brief handles this differently (conditional rendering in App.tsx), but the dedicated component approach is cleaner for larger apps. 

// ProtectedRoute.tsx — reusable guard component
import { Navigate, Outlet } from "react-router-dom";
import { useAppSelector } from "../app/hooks";
import { selectIsLoggedIn } from "../features/auth/authSlice/authSelectors";

export default function ProtectedRoute({ requiredRole }: { requiredRole?: string }) {
  const isLoggedIn = useAppSelector(selectIsLoggedIn);
  const user       = useAppSelector(selectUser);

  if (!isLoggedIn) {
    return <Navigate to="/login" replace />;
  }

  if (requiredRole && user?.role !== requiredRole) {
    return <Navigate to="/" replace />; // wrong role → home
  }

  return <Outlet />; // ← renders the child route if checks pass
}

// App.tsx — using ProtectedRoute
<Routes>
  <Route path="/login"    element={<Login    />} />
  <Route path="/register" element={<Register />} />

  {/* All authenticated users */}
  <Route element={<ProtectedRoute />}>
    <Route path="/"          element={<Home       />} />
    <Route path="/profile"   element={<Profile    />} />
  </Route>

  {/* ADMIN only */}
  <Route element={<ProtectedRoute requiredRole="ADMIN" />}>
    <Route path="/briefs"    element={<BriefsDashboard />} />
  </Route>
</Routes>

404 Not Found Route

A route with path="*" matches anything that no other route matched. Place it last. 

function NotFound() {
  const navigate = useNavigate();
  return (
    <div className="text-center py-20">
      <h1>404 — Page Not Found</h1>
      <button onClick={() => navigate("/")}>Go Home</button>
    </div>
  );
}

// In Routes — must be last
<Routes>
  <Route path="/"      element={<Home     />} />
  <Route path="/briefs" element={<Briefs   />} />
  <Route path="*"      element={<NotFound />} />  {/* ← must be last */}
</Routes>

Passing State Between Routes

Sometimes you need to pass data from one page to another without putting it in the URL (it might be sensitive, or too complex for a query string). Use location.state.

Register page
has email + rememberMe
→
navigate("/verify-code",
{ state: { email, rememberMe } })
→
VerifyCode page
reads email from location.state
Register.tsx — sending state
const navigate = useNavigate();

// After registration succeeds
navigate("/verify-code", {
  state: { email: formData.email, rememberMe: formData.rememberMe }
});
VerifyCode.tsx — receiving state
const location = useLocation();

// Cast to the expected shape — state is typed as unknown
const email = (location.state as { email?: string; rememberMe?: boolean })?.email || "";
const rememberMe = (location.state as { email?: string; rememberMe?: boolean })?.rememberMe ?? false;

// Guard: if no email, user came here directly (not from register) — send them back
useEffect(() => {
  if (!email) navigate("/register");
}, [email, navigate]);
State is lost on refresh location.state is stored in the browser's session history. If the user refreshes the page, location.state becomes null. That's why VerifyCode guards against it and redirects to /register.

Programmatic Navigation Patterns

// 1. Navigate after async action succeeds
const handleSubmit = async () => {
  const result = await dispatch(createBriefThunk(data));
  if (createBriefThunk.fulfilled.match(result)) {
    navigate("/my-briefs"); // only navigate on success
  }
};

// 2. Navigate after deletion
const handleDelete = async () => {
  await dispatch(deleteBriefThunk(id));
  navigate(-1); // go back to the previous page
};

// 3. Navigate on logout — go to landing page
function handleLogout() {
  dispatch(logout());
  navigate("/");
}

// 4. Replace — don't add to history (user can't go back)
navigate("/home", { replace: true }); // useful after login/logout

// 5. Navigate with state
navigate("/verify-code", { state: { email: "omar@..." } });

Conditional Routes (Smart Brief approach)

Smart Brief's App.tsx uses a simpler alternative to ProtectedRoute: directly render a different component for the same path based on login state. 

App.tsx
const isLoggedIn = useAppSelector(selectIsLoggedIn);

<Routes>
  {/* "/" shows Home for logged-in users, Landing for guests */}
  {isLoggedIn ? (
    <Route path="/" element={<Home    />} />
  ) : (
    <Route path="/" element={<Landing />} />
  )}
  {/* Other routes are always available */}
  <Route path="/login"    element={<Login    />} />
  <Route path="/register" element={<Register />} />
  <Route path="/briefs/:id" element={<BriefDetails />} />
</Routes>

Full App.tsx Routing — Annotated

App.tsx
export default function SmarBriefApp() {
  const dispatch = useAppDispatch();
  const location     = useLocation();       // reads current URL path
  const [searchParams, setSearchParams] = useSearchParams(); // ?key=val
  const token      = useAppSelector(selectToken);
  const user        = useAppSelector(selectUser);
  const isLoggedIn  = useAppSelector(selectIsLoggedIn);

  // ① Detect if on an auth page (to swap background illustration)
  const authPaths  = ['/register', '/login', '/verify-code', '/forgot-password', '/reset-password'];
  const isAuthPage = authPaths.includes(location.pathname);

  // ② Handle Google OAuth redirect: /?token=... → save token → clean URL
  useEffect(() => {
    const googleToken = searchParams.get("token");
    if (googleToken) {
      localStorage.setItem("token", googleToken);
      dispatch(setToken(googleToken));
      const next = new URLSearchParams(searchParams.toString());
      next.delete("token");
      setSearchParams(next, { replace: true });
    }
  }, [dispatch, searchParams, setSearchParams]);

  // ③ Restore session on page refresh
  useEffect(() => {
    if (token && !user) dispatch(getMeThunk());
  }, [dispatch, token, user]);

  return (
    <div>
      <Navbar />
      <Routes>
        {/* ④ Home = different component based on login state */}
        {isLoggedIn
          ? <Route path="/" element={<Home    />} />
          : <Route path="/" element={<Landing />} />
        }
        <Route path="/new-project"    element={<NewProject      />} />
        <Route path="/my-briefs"       element={<MyBriefs        />} />
        <Route path="/briefs"           element={<BriefsDashboard />} />
        <Route path="/assigned-briefs" element={<BriefsDashboard />} />
        <Route path="/briefs/:id"       element={<BriefDetails    />} />
        <Route path="/register"         element={<Register        />} />
        <Route path="/verify-code"      element={<VerifyCode      />} />
        <Route path="/login"            element={<Login           />} />
        <Route path="/forgot-password" element={<ForgotPassword />}  />
        <Route path="/reset-password"  element={<ResetPassword  />}  />
      </Routes>
      <Footer />
    </div>
  );
}

// Route: /briefs/:id
