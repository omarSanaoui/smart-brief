Tailwind CSS

Everything you need to work with Tailwind — every utility, every modifier, v4 setup, and real Smart Brief examples.

What is Tailwind?

Tailwind CSS is a utility-first CSS framework. Instead of writing CSS in a separate file, you apply small single-purpose classes directly to your HTML/JSX.

Traditional CSS

/* button.css */
.btn {
  background: #414CC4;
  color: white;
  padding: 12px 24px;
  border-radius: 8px;
  font-weight: 700;
  transition: background 0.2s;
}
.btn:hover {
  background: #3a44b0;
}
/* button.jsx */
<button className="btn">Click</button>

Tailwind

/* No CSS file needed */
<button
  className="bg-[#414CC4] text-white
    px-6 py-3 rounded-lg font-bold
    hover:bg-[#3a44b0] transition-colors"
>
  Click
</button>

All styling is inline in the JSX. No CSS context-switching.

Tailwind scans your JSX files, finds every class you use, and generates only those CSS rules — the final CSS file is tiny even for large projects.

Utility-First Concept

Each class does one thing:

ClassCSS it generates
flexdisplay: flex
items-centeralign-items: center
gap-4gap: 1rem
text-whitecolor: #fff
font-boldfont-weight: 700
rounded-xlborder-radius: 0.75rem
hover:bg-blue-500&:hover { background-color: #3b82f6 }
md:flex@media (min-width:768px) { display: flex }

You combine classes to build complex styles — no naming, no file switching, no specificity battles.

Setup (Tailwind v4)

# Install
npm install tailwindcss @tailwindcss/vite

In vite.config.ts — add the Tailwind plugin:

import tailwindcss from '@tailwindcss/vite'
import { defineConfig } from 'vite'

export default defineConfig({
  plugins: [react(), tailwindcss()]
})

In your main CSS file (index.css) — one line replaces all v3 directives:

/* index.css — that's it, no tailwind.config.js needed */
@import "tailwindcss";

Import that CSS in your entry point (main.tsx):

import "./index.css"

group & group-hover:

The group modifier lets a parent's hover state style its children. Add group to the parent, then group-hover: to any child. 

/* BriefCard — hovering the card scales the icon and changes title color */
<div className="group bg-[#1A2238] hover:border-sbteal/50 rounded-2xl p-6 transition-all">

  {/* Icon — scales up when PARENT is hovered */}
  <div className="bg-[#2D3652] p-3 rounded-xl group-hover:scale-110 transition-transform">
    <FileText />
  </div>

  {/* Title — turns teal when PARENT is hovered */}
  <h3 className="text-xl font-bold group-hover:text-sbteal transition-colors">
    {brief.title}
  </h3>

  {/* Arrow — slides right when PARENT is hovered */}
  <ChevronRight className="group-hover:translate-x-1 transition-transform" />
</div>

Named groups

If you have nested groups, name them to avoid ambiguity:

/* group/card and group/button are separate groups */
<div className="group/card">
  <button className="group/button">
    <span className="group-hover/card:text-blue-500">card hovered</span>
    <span className="group-hover/button:text-red-500">button hovered</span>
  </button>
</div>

peer & peer-*

Like group, but for sibling elements. Add peer to an element, then peer-hover:, peer-focus:, peer-checked: on a sibling that comes after it. 

/* Show label when input is focused */
<input className="peer border ..." />
<label className="text-gray-400 peer-focus:text-blue-500 transition-colors">
  Email
</label>

/* Toggle visibility based on checkbox state */
<input type="checkbox" className="peer hidden" id="toggle" />
<label htmlFor="toggle">Show more</label>
<div className="hidden peer-checked:block">
  Hidden content revealed when checked
</div>

placeholder:

/* Style placeholder text */
<input
  placeholder="Enter your email..."
  className="placeholder-white/40 text-white bg-[#2D3652]"
/>

/* On focus, placeholder fades out */
<input className="placeholder-gray-400 focus:placeholder-gray-300" />

dark:

Apply styles only when the user's OS/browser is in dark mode:

<div className="bg-white text-black dark:bg-gray-900 dark:text-white">
  Adapts to system preference
</div>
Smart Brief doesn't use dark: — it's always dark The entire app uses dark backgrounds by default. You'd need dark: only if you wanted a light mode that switches to dark.

Custom Theme (v4)

Define your own colors, fonts, and spacing in the @theme block in your CSS file:

/* index.css — Smart Brief's @theme */
@import "tailwindcss";

@theme {
  /* Custom font — use as font-poppins */
  --font-poppins: ['Poppins', 'sans-serif'];

  /* Custom colors — use as bg-theme, text-theme, etc. */
  --color-theme:    #141824;   /* bg-theme */
  --color-sbpurple: #414CC4;   /* bg-sbpurple, text-sbpurple, border-sbpurple */
  --color-sbteal:   #67CFB1;   /* bg-sbteal, text-sbteal, border-sbteal */
}

After this, you can use these as any Tailwind color utility:

bg-theme           // background-color: #141824
text-sbteal        // color: #67CFB1
border-sbpurple    // border-color: #414CC4
bg-sbteal/10       // + opacity modifier works too!
font-poppins       // font-family: 'Poppins'

Adding more custom values

@theme {
  /* Custom spacing */
  --spacing-18: 4.5rem;       /* p-18, m-18, gap-18 */

  /* Custom breakpoint */
  --breakpoint-xs: 480px;     /* xs:flex, xs:hidden, etc. */

  /* Custom font size */
  --text-xxs: 0.625rem;       /* text-xxs */

  /* Custom border radius */
  --radius-4xl: 2rem;         /* rounded-4xl */

  /* Custom animation */
  --animate-fade-in: fadeIn 0.3s ease-out;
}

Applying CSS Directly — @apply

If you have a class that's reused many times with the same long Tailwind string, you can create a CSS class using @apply: 

/* index.css */
@layer components {
  .btn-primary {
    @apply bg-sbpurple text-white px-6 py-3 rounded-xl font-bold
           hover:bg-[#3a44b0] transition-colors disabled:opacity-50
           disabled:cursor-not-allowed;
  }

  .input-field {
    @apply bg-[#2D3652] border border-[#2E3A5C] rounded-md px-4 py-3
           text-white placeholder-white/40 text-sm focus:outline-none
           focus:border-[#414CC4] transition-colors;
  }

  .card {
    @apply bg-[#1A2238] border border-[#2E3A5C] rounded-2xl p-6;
  }
}
/* Now use in JSX */
<button className="btn-primary">Submit</button>
<input className="input-field" />
When to use @apply vs inline classes Use @apply for component styles you repeat across many files (buttons, inputs, cards). Keep one-off styles inline. Don't @apply everything — you lose the benefits of utility-first.

Smart Brief Patterns

The status badge pattern

/* StatusBadge.tsx — shared base classes + status-specific color classes */
const base = "flex items-center gap-1.5 font-bold uppercase tracking-wider rounded-full"
const size = large ? "px-5 py-2.5 text-xs" : "px-3 py-1 text-[10px]"

// Each status gets text + bg + border all from the same color
case 'ACCEPTED': return <span className={`${base} ${size} text-sbteal bg-sbteal/10 border border-sbteal/20`}>

case 'REFUSED':  return <span className={`${base} ${size} text-rose-400 bg-rose-400/10 border border-rose-400/20`}>

Conditional classes pattern

/* BriefCard — conditional cursor and bg based on prop */

Dynamic Class Merging

Template literals get messy fast. Use the clsx or cn utility for clean conditional classes:

# Install clsx
npm install clsx
import clsx from "clsx"

// clsx merges class strings, conditionally includes classes
<div className={clsx(
  "bg-[#1A2238] border rounded-2xl p-6 transition-all",
  onClick && "cursor-pointer hover:bg-[#1f2845]",
  isActive && "border-sbteal",
  !isActive && "border-[#2E3A5C]"
)} />

// or with object syntax
clsx({
  "opacity-50 cursor-not-allowed": disabled,
  "hover:bg-sbpurple":             !disabled,
})

cn utility (clsx + tailwind-merge)

If you apply conflicting classes (e.g. both p-4 and p-6), Tailwind generates both but only the last in the stylesheet wins — not the last in your className string. tailwind-merge fixes this:

npm install clsx tailwind-merge
// lib/utils.ts — standard pattern in most React+Tailwind projects
import { clsx, type ClassValue } from "clsx"
import { twMerge } from "tailwind-merge"

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs))
}

// Usage — p-6 wins over p-4, no conflict
cn("p-4 text-sm", "p-6")  // → "text-sm p-6"
